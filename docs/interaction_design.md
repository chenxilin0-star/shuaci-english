# CET-4/CET-6 微信小程序 — 完整交互设计方案

> 文档版本：v1.0  
> 适用平台：微信小程序  
> 目标用户：英语四六级备考学生  
> 设计原则：轻量化、即时反馈、游戏化激励、断点续学

---

## 1. 用户学习动线设计

### 1.1 新用户首次使用流程（冷启动路径）

```
[打开小程序] → [闪屏/品牌页 1.5s] → [授权登录弹窗] → [选择目标考试 CET-4/CET-6]
    ↓
[选择词库（核心词/高频词/全部）] → [设定每日学习目标（默认20词）] → [进入首页仪表盘]
    ↓
[新用户引导浮层（首次学习引导）] → [进入单词卡片学习]
```

| 步骤 | 页面/模块 | 交互说明 | 设计要点 |
|------|----------|----------|----------|
| 1 | 闪屏页 | 展示品牌Logo + Slogan，1.5s自动跳转 | 不可跳过，强化品牌认知 |
| 2 | 授权登录 | 微信一键登录按钮，需用户主动点击 | 明确告知"仅获取昵称/头像"，降低顾虑 |
| 3 | 目标考试选择 | 两个大卡片选项 CET-4 / CET-6 | 默认不选中，必须选择后才能继续；选择后有微动效 |
| 4 | 词库选择 | 单选列表：核心词汇（500词）/ 高频词汇（1000词）/ 完整词库（4500词） | 推荐标签标注"最多人选择"，默认选中核心词汇 |
| 5 | 每日目标 | 滑动选择器：10/20/30/50词 | 默认20词；标注"每天10分钟"等时间参考 |
| 6 | 进入首页 | 展示学习仪表盘 + 今日任务入口 | 首次进入显示欢迎Toast"目标已设定，开始学习吧！" |
| 7 | 新手引导 | 3步蒙层引导：① 点击开始学习 ② 卡片翻转查看释义 ③ 收藏按钮位置 | 可点击"跳过"；最后一步引导进入学习 |

**流失干预点**：
- 在授权登录页停留>5s未操作：底部弹出"无需手机号，1秒登录即可学习"
- 在目标选择页停留>8s：自动播放选择推荐动效
- 任意步骤点击返回：二次确认"退出后需重新开始，确认退出？"

### 1.2 每日学习常规流程

```
[打开小程序] → [首页仪表盘加载] → [自动签到检测/弹窗]
    ↓
[继续上次学习] 或 [今日新任务] 或 [选择其他模块]
    ↓
[单词卡片学习 → 拼写练习 → 听写练习]（根据当日计划组合）
    ↓
[完成今日任务] → [结果页展示] → [分享/打卡]
    ↓
[查看学习统计] 或 [进入错题本复习] 或 [退出]
```

| 时间节点 | 用户行为 | 系统响应 | 交互设计 |
|----------|----------|----------|----------|
| 打开小程序 | 冷启动/热启动 | 加载本地缓存数据→请求服务器同步 | 冷启动显示骨架屏；热启动直接展示数据 |
| 首页加载完成 | 浏览仪表盘 | 检测今日是否已签到 | 未签到：自动弹出签到弹窗（非全屏，可关闭） |
| 点击"继续学习" | 恢复上次进度 | 读取localStorage中的lastStudyPosition | 显示"上次学习到第X词"，1.5s后自动进入 |
| 点击"今日新任务" | 开始新一组单词 | 根据每日目标生成今日词单 | 显示"今日目标：20词"确认浮层 |
| 学习过程中 | 翻转卡片、收藏、发音 | 实时记录操作到本地缓存队列 | 1秒防抖后同步到服务器 |
| 完成一组单词 | 学习结束 | 弹出结果页：学习数量、正确率、用时 | 显示"完成今日目标"激励文案 |
| 结果页关闭 | 返回首页 | 更新首页学习数据，连续天数+1（如首次完成） | 首页数据刷新动画 |

### 1.3 进阶学习动线（薄弱点闭环）

```
[日常学习中发现错词/模糊词] → [自动加入错题本/生词本]
    ↓
[首页"薄弱点提醒"入口高亮] → [进入专项复习]
    ↓
[错题本筛选"高频错题"] → [重新练习模式（仅错题）]
    ↓
[练习完成 → 正确率提升反馈] → [标记为"已掌握" 或 继续复习]
    ↓
[达成阶段目标] → [成就解锁] → [分享成就]
```

| 阶段 | 触发条件 | 用户路径 | 激励反馈 |
|------|----------|----------|----------|
| 发现薄弱点 | 单词3次答错/拼写错误/收藏 | 自动加入错题本 + 首页显示"有X个词需要复习" | 红点提醒 + 弱提醒Toast |
| 专项复习 | 用户点击首页提醒/主动进入错题本 | 错题本 → 筛选"待复习" → 选择"重新练习" | 练习结果对比上次正确率 |
| 巩固掌握 | 错题连续答对2次 | 弹窗提示"已掌握，从错题本移除？" | 绿色勾选 + 鼓励文案 |
| 阶段达成 | 连续7天完成目标 / 掌握500词 | 全屏成就动画 + 解锁新成就 | 全屏confetti动效 + 分享卡片 |

### 1.4 用户流失风险点的交互干预设计

| 风险场景 | 流失概率 | 干预时机 | 干预手段 | 交互实现 |
|----------|----------|----------|----------|----------|
| 首次打开未授权登录 | 高(40%) | 停留>5s | 底部提示条 | "微信一键登录，无需手机号" + 显眼的登录按钮 |
| 授权后未选择目标 | 中(25%) | 停留>8s | 推荐动效 | 自动高亮推荐选项 + 文字引导"CET-4考生最多选择" |
| 学习3天后中断 | 高(35%) | 第4天打开 | 推送召回 + 首页特殊提示 | 首页横幅"你已经学了3天，中断太可惜！" + 火焰图标变灰 |
| 单次学习<2分钟退出 | 中(20%) | 点击退出 | 确认弹窗 + 保留进度 | "只学了2分钟，确定退出？进度已保存" |
| 连续错题挫败感 | 高(30%) | 连续3题错误 | 鼓励干预 | 弹出"这些词确实比较难，加入错题本慢慢攻克？" |
| 完成目标后无后续 | 中(15%) | 结果页关闭 | 下一步引导 | "再来一组？" / "去复习错题？" / "查看今日成就" |
| 7天未打开小程序 | 高(50%) | 服务端推送 | 模板消息推送 | "你的单词们想你了，今天只要5分钟" + 直达学习页 |
| 深色模式/设置入口难找 | 低(5%) | 用户长按首页 | 快捷菜单 | 首页长按弹出快捷设置浮层 |

## 2. 单词学习交互

### 2.1 卡片学习完整交互流程

```
[进入学习] → [加载词单] → [展示第一张卡片正面]
    ↓
[用户操作：点击卡片/滑动/点击翻转按钮]
    ↓
[卡片翻转180°展示背面：释义+例句+发音按钮+收藏按钮]
    ↓
[用户操作：点击发音 / 点击收藏 / 直接翻页]
    ↓
[滑动/点击"下一个" → 卡片滑出 → 下一张卡片滑入]
    ↓
[循环直到词单完成] → [显示学习结果页]
```

#### 状态机：卡片学习

| 当前状态 | 触发事件 | 下一状态 | 系统反馈 | 用户可执行操作 |
|----------|----------|----------|----------|----------------|
| 初始化 | 进入学习页 | 加载中 | 显示骨架屏/加载动画 | 等待 |
| 加载中 | 数据加载完成 | 正面展示 | 正面朝上展示单词+音标 | 翻转、滑动、收藏、发音 |
| 正面展示 | 点击卡片/翻转按钮 | 翻转动画中 | 卡片180°Y轴旋转动画(300ms) | 不可操作（动画中锁定） |
| 翻转动画中 | 动画完成 | 背面展示 | 展示释义+例句+发音+收藏 | 发音、收藏、翻页 |
| 背面展示 | 点击发音按钮 | 播放中 | 按钮变为波形动画 | 不可重复点击（连点防重） |
| 背面展示 | 点击收藏按钮 | 已收藏/取消收藏 | 心形图标填充动画 + Toast | 继续其他操作 |
| 背面展示 | 向左滑动/点击"下一个" | 翻页动画中 | 当前卡片左滑退出，新卡片右滑进入 | 滑动中可中断 |
| 翻页动画中 | 动画完成 | 正面展示（新卡片） | 更新进度条 | 翻转、滑动... |
| 正面展示 | 向右滑动 | 返回上一张 | 卡片右滑退出，上一张左滑进入 | 仅限非第一张 |
| 任意状态 | 点击"退出"按钮 | 退出确认 | 弹出确认弹窗 | 确认/取消 |
| 任意状态 | 接听电话/退后台 | 后台暂停 | 记录离开时间戳 | 无 |
| 后台暂停 | 返回前台 | 恢复学习 | 显示"欢迎回来" + 当前卡片 | 继续操作 |

---

### 2.2 翻页手势设计

| 参数 | 设定值 | 说明 |
|------|--------|------|
| 触发阈值 | 水平滑动 > 60px | 小于此值视为点击，不触发翻页 |
| 灵敏度 | 滑动速度 > 0.3px/ms 或 距离 > 60px | 满足任一条件即触发 |
| 垂直容忍 | 垂直偏移 < 40px | 垂直滑动超过此值视为滚动，取消水平翻页 |
| 翻页动画时长 | 280ms | ease-out曲线 |
| 边界反馈 | 首/尾卡片滑动时阻尼效果（最多偏移30px后回弹） | 配合触觉反馈（如支持） |
| 防误触 | 卡片翻转后 200ms 内不响应滑动 | 避免翻转手势与滑动手势冲突 |
| 滑动方向 | 左滑=下一个，右滑=上一个 | 符合阅读习惯 |

### 2.3 卡片翻转设计

| 参数 | 设定 | 说明 |
|------|------|------|
| 触发方式 | 点击卡片任意位置 或 点击"翻转"按钮 | 两种方式并存，用户可选择 |
| 翻转动画 | CSS 3D transform rotateY(0→180deg) | 正面visible背面hidden，翻转后切换 |
| 动画时长 | 300ms | cubic-bezier(0.4, 0.0, 0.2, 1) |
| 动画中可中断 | 否 | 翻转动画300ms内锁定，防止快速连点导致状态混乱 |
| 翻转后默认展示 | 中文释义 + 英文例句 + 发音按钮 + 收藏按钮 | 例句默认展开，可折叠 |
| 自动翻转 | 不支持 | 需用户主动触发，加深记忆 |

### 2.4 发音按钮交互

| 状态 | 视觉表现 | 交互行为 |
|------|----------|----------|
| 默认 | 灰色扬声器图标 | 点击触发TTS播放 |
| 播放中 | 蓝色扬声器图标 + 声波波纹动画 | 点击停止播放；播放完成后自动恢复默认 |
| 连点防重 | 播放中按钮置灰，300ms冷却 | 连续点击仅响应第一次 |
| 播放失败 | 图标变红 + Toast"播放失败，点击重试" | 点击后重新初始化TTS并播放 |
| 网络断开 | 图标置灰 + Toast"无网络，发音暂不可用" | 本地无缓存音频时不播放 |

### 2.5 收藏交互

| 步骤 | 用户操作 | 系统反馈 | 数据操作 |
|------|----------|----------|----------|
| 1 | 点击心形收藏按钮 | 心形图标填充变色（灰→红）+ 微缩放动画 | 加入用户收藏表（本地即时+服务器同步） |
| 2 | - | Toast"已收藏"，1.5s自动消失 | - |
| 3 | 再次点击已收藏按钮 | 心形图标变空心（红→灰） | 从收藏表移除 |
| 4 | - | Toast"已取消收藏" | - |

**边界情况**：
- 未登录用户点击收藏：弹出登录弹窗 → 登录后继续执行收藏操作
- 断网时收藏：本地缓存操作，恢复网络后批量同步，冲突时以服务器为准

### 2.6 进度保存策略

| 保存时机 | 策略 | 说明 |
|----------|------|------|
| 每学完1个单词 | 本地缓存学习状态 | 包括：单词ID、是否翻转过、是否收藏、是否发音 |
| 翻页后1秒 | 防抖同步服务器 | 使用防抖（debounce 1000ms），避免频繁请求 |
| 退出学习页 | 强制同步服务器 | onUnload生命周期触发，确保进度不丢失 |
| 退后台（切微信） | 本地缓存+时间戳 | 记录离开时间，返回后恢复 |
| 网络恢复 | 批量同步本地队列 | 检查localStorage中的pendingSync队列 |

**断网本地缓存策略**：
```
localStorage结构：
{
  "studyProgress": {
    "userId": "xxx",
    "bookId": "cet4_core",
    "currentIndex": 15,
    "totalWords": 20,
    "wordRecords": [
      {"wordId": "abandon", "flipped": true, "starred": false, "pronounced": true},
      ...
    ],
    "lastStudyTime": 1699123456789,
    "pendingSync": true
  }
}
```

**恢复时冲突处理**：
| 场景 | 冲突检测 | 解决策略 |
|------|----------|----------|
| 本地进度 > 服务器进度 | 比较currentIndex | 以本地为准，同步到服务器 |
| 本地进度 < 服务器进度 | 比较currentIndex | 以服务器为准，提示"已从第X词继续" |
| 本地有服务器没有的收藏 | 对比收藏列表 | 合并去重后同步 |
| 多端同时学习 | 服务器记录最后更新时间 | 以最新时间为准，覆盖旧数据 |

### 2.7 拼写练习交互流程

```
[进入拼写练习] → [展示题目：中文释义 + 音标 + 词性]
    ↓
[用户输入拼写] → [点击提交 或 按回车]
    ↓
[系统判定] → [正确：绿色√ + 自动下一题（1.5s）] 或 [错误：红色× + 正确答案 + 手动继续]
    ↓
[完成所有题目] → [结果页：正确率% + 错题列表 + 重新练习/返回]
```

#### 状态机：拼写练习

| 当前状态 | 触发事件 | 下一状态 | 系统反馈 |
|----------|----------|----------|----------|
| 初始化 | 进入练习 | 题目展示 | 展示释义+输入框+提交按钮 |
| 题目展示 | 用户输入字符 | 输入中 | 实时显示输入内容 |
| 输入中 | 点击提交 / 回车键 | 判定中 | 按钮loading状态（200ms） |
| 判定中 | 答案正确 | 正确反馈 | 输入框绿色边框 + 绿色√图标 + 播放正确音效 |
| 判定中 | 答案错误 | 错误反馈 | 输入框红色边框 + 红色×图标 + 显示正确答案（灰色） |
| 正确反馈 | 1.5s定时器/点击 | 下一题展示 | 自动进入下一题；最后一题进入结果页 |
| 错误反馈 | 用户点击"继续" | 下一题展示 | 手动确认后进入下一题 |
| 结果页 | 点击"重新练习" | 初始化 | 仅错题 / 全部题目 |
| 结果页 | 点击"返回首页" | 退出 | 返回首页仪表盘 |

**输入细节**：
| 参数 | 设定 | 说明 |
|------|------|------|
| 键盘类型 | 英文键盘 | input type="text"，避免中文输入法 |
| 大小写处理 | 不敏感 | "Apple"和"apple"都判定正确 |
| 自动聚焦 | 进入题目后自动聚焦输入框 | 软键盘自动弹出 |
| 自动提交 | 输入字符数 = 目标单词长度时，可自动提交（需开启设置） | 默认关闭，避免误提交 |
| 连字符/空格 | 忽略差异 | "co-operate"和"cooperate"视为相同（可配置） |
| 清空功能 | 输入框右侧显示"×"清空按钮 | 点击清空所有输入 |
| 提示功能 | 点击"提示"按钮显示首字母 | 使用提示后该题不计入正确率 |

### 2.8 听写练习交互流程

```
[进入听写练习] → [自动播放第一个单词TTS]
    ↓
[用户听到发音 → 根据中文提示 → 输入单词]
    ↓
[逐字母实时检查] → [全部输入完成后自动判定] 或 [点击提交]
    ↓
[正确：绿色√ + 自动播放下一词] 或 [错误：红色× + 重播按钮 + 显示正确拼写]
    ↓
[完成所有听写] → [结果页]
```

**听写特有交互**：
| 功能 | 交互方式 | 说明 |
|------|----------|------|
| 自动播放 | 进入题目后自动播放2次（间隔2s） | 播放按钮变为"播放中"状态 |
| 手动重播 | 点击播放按钮无限重播 | 无冷却时间 |
| 暂停/继续 | 点击暂停按钮暂停播放 | 暂停后显示"继续"按钮 |
| 中文提示 | 题目区域显示中文释义 | 始终可见 |
| 逐字母检查 | 输入每个字母实时比对 | 正确字母绿色显示，错误字母红色+震动反馈 |
| 进度控制 | 顶部进度条 + 可拖拽 | 拖拽跳转到指定题目 |


## 3. 阅读答题交互

### 3.1 阅读文章展示策略

| 方案 | 决策 | 说明 |
|------|------|------|
| 分页 or 滚动 | **连续滚动** | CET阅读文章长度适中（300-500词），滚动比分页更流畅 |
| 题目位置 | **底部悬浮题目栏** | 文章占上部70%，底部30%展示当前题目，边读边答 |

## 8. 异常与边界情况

### 8.1 网络断开处理

| 场景 | 检测方式 | 系统行为 | 用户反馈 |
|------|----------|----------|----------|
| 进入页面无网 | 请求失败 | 显示缓存数据/空状态 | Toast"网络断开，显示离线内容" |
| 学习中断网 | 同步请求失败 | 本地缓存操作，加入pending队列 | Toast"网络断开，进度已本地保存" |
| 翻页同步失败 | 防抖同步失败 | 标记pending，下次联网批量同步 | 无反馈（后台处理） |
| 收藏同步失败 | 请求失败 | 本地标记，加入pending队列 | Toast"收藏已保存，联网后同步" |
| 网络恢复 | 系统事件监听 | 自动触发批量同步 | Toast"网络已恢复，数据已同步" |

**本地缓存操作队列**：
```javascript
// 操作队列结构
pendingQueue: [
  {type: 'STUDY_PROGRESS', data: {...}, timestamp: 1699123456},
  {type: 'STAR_WORD', wordId: 'abandon', action: 'add', timestamp: 1699123460},
  {type: 'STUDY_PROGRESS', data: {...}, timestamp: 1699123470}
]
// 恢复网络后按timestamp顺序批量执行
```

**冲突处理**：
| 冲突场景 | 解决策略 |
|----------|----------|
| 本地收藏，服务器未收藏 | 以本地为准，同步到服务器 |
| 本地未收藏，服务器已收藏 | 弹窗"该词在另一设备已收藏，是否同步？" |
| 本地进度>服务器进度 | 以本地为准 |
| 多端同时学习同一词库 | 以最新timestamp为准，合并学习记录 |

| 场景 | 同步策略 | 冲突处理 |
|------|----------|----------|

## 9. 状态与反馈系统

### 9.1 Toast提示使用规范

| 类型 | 使用场景 | 显示时长 | 视觉样式 | 示例文案 |
|------|----------|----------|----------|----------|
| **成功** | 操作成功、收藏、加入生词本 | 1.5s | 绿色背景 + 白色✓图标 | "已收藏" / "已加入错题本" / "进度已保存" |
| **失败** | 操作失败、网络错误 | 2.5s | 红色背景 + 白色✗图标 | "操作失败，请重试" / "网络断开" |
| **信息** | 提示信息、引导 | 2.0s | 灰色背景 + 白色ℹ图标 | "已开始播放" / "已跳过" / "欢迎回来" |
| **警告** | 需要注意但不阻断 | 2.0s | 橙色背景 + 白色⚠图标 | "错题本已满1000条，建议清理" |

**Toast使用规则**：
- 同一时刻只显示1个Toast，新的覆盖旧的
- Toast出现时页面可继续操作，非模态
- 位置：页面垂直居中偏下（距离底部25%）
- 动画：淡入200ms + 停留 + 淡出200ms
- 最长显示3秒，避免长时间遮挡

## 附录A：交互设计原则速查表

| 原则 | 描述 | 应用场景 |
|------|------|----------|
| **即时反馈** | 用户操作后200ms内给出视觉反馈 | 按钮点击、收藏、选择 |
| **防错设计** | 危险操作需二次确认，提供撤销 | 删除、清空、重置 |
| **断点续传** | 任何中断都可恢复，进度不丢失 | 学习、答题、练习 |
| **容错设计** | 网络异常有降级方案 | 离线缓存、默认数据 |
| **操作合并** | 批量操作优于逐条操作 | 收藏、删除、标记 |
| **渐进披露** | 复杂信息分层展示 | 例句展开、解析查看 |
| **游戏化反馈** | 正向激励即时可见 | 打卡、成就、正确率 |
| **手势一致性** | 全产品手势逻辑统一 | 滑动翻页、返回 |
