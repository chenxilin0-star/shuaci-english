## 一、项目概述与核心决策

### 1.1 产品定义

**产品名称**: 刷词英语（小程序版）
**目标用户**: 18-25岁大学生群体，备考CET-4/CET-6
**核心价值**: 提供系统化的四六级词汇学习、语法讲解和阅读答题训练
**商业模式**: 前期完全免费，用户微信登录即可使用全部功能；后续可扩展付费增值服务

### 1.2 功能模块规划

| 模块 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| 首页/仪表盘 | 学习数据概览、词库入口 | P0 | 核心入口 |
| 单词学习 | 卡片翻转学习、拼写练习 | P0 | 核心功能 |
| 语法学习 | 语法知识点、例句讲解 | P0 | 四六级核心语法 |
| 阅读答题 | 阅读理解练习、真题训练 | P0 | 四六级核心题型 |
| 生词本/收藏 | 个人单词管理 | P1 | 学习辅助 |
| 错题本 | 错题管理与复习 | P1 | 学习辅助 |
| 学习统计 | 签到、时长、正确率 | P1 | 激励体系 |
| 个人中心 | 设置、用户信息 | P1 | 基础功能 |
| 视频课程 | 视频课程学习 | P2 | 后续扩展 |

### 1.3 核心技术决策

| 决策项 | 决策结论 | 理由 |
|--------|----------|------|
| 技术栈 | 微信小程序原生 + 云开发 | 免运维、快速上线、成本可控 |
| 认证方式 | 微信一键登录 | 无需注册、降低使用门槛 |
| 是否收费 | 前期完全免费 | 快速获取用户、验证产品 |
| 词汇数据源 | ECDICT开源词库 + COCA词频 | 免费、可商用、覆盖全面 |
| TTS方案 | Edge-TTS(免费) + 云存储缓存 | 免费神经语音、质量好 |
| AI功能 | MVP阶段不引入 | 降低复杂度、控制成本 |
| 深色模式 | 支持 | 跟随系统+手动切换 |

## 二、技术架构方案

> **详细技术架构文档**: 见 `tech_architecture.md`（2146行完整方案）

### 2.1 云开发可行性结论

**微信小程序云开发能支撑本项目90%以上的技术需求**，推荐采用。关键指标：

| 指标 | 结论 |
|------|------|
| 开发效率 | 免运维、前后端一体，MVP仅需2-4周 |
| 初期成本 | 1万用户以内基本在免费额度内 |
| 扩展路径 | 日活超过1万后可评估迁移自建后端 |
| 主要限制 | 无多表事务、全文检索能力有限（可通过设计规避） |

**推荐策略**: MVP阶段（0-6个月）采用云开发快速验证，用户增长期再评估是否迁移。

### 2.2 数据库核心设计

共18个集合，分为静态数据、用户数据、系统数据三大类：

**静态数据**（初始化后很少变动）：

| 集合 | 预估数据量 | 说明 |
|------|-----------|------|
| `words` | ~10,000条 | CET-4约4500词 + CET-6约5500词 |
| `sentences` | ~2,000条 | 阅读理解相关句子 |
| `grammar_topics` | ~100条 | 四六级高频语法知识点 |
| `reading_passages` | ~200条 | 四六级阅读真题/模拟题 |
| `reading_questions` | ~1,000条 | 每篇文章约5题 |

**用户数据**（随用户增长）：

| 集合 | 预估数据量(1万用户) | 说明 |
|------|-------------------|------|
| `users` | 10,000条 | 用户基本信息与设置 |
| `user_progress` | 10,000条 | 每用户每词库一条学习进度 |
| `user_word_records` | 500,000条 | 学习过的单词状态记录 |
| `mistake_books` | 50,000条 | 错题记录 |
| `favorites` | 50,000条 | 收藏记录 |
| `checkins` | 300,000条 | 签到记录 |

### 2.3 云函数模块划分

共需约18个云函数，按模块分组：

| 模块 | 云函数 | 职责 |
|------|--------|------|
| 认证 | `authLogin`, `authLogout` | 微信登录、登出 |
| 词库 | `getWordBanks`, `getWordList`, `getWordDetail` | 词库/单词查询 |
| 学习进度 | `updateWordProgress`, `getStudyProgress` | 进度保存/恢复 |
| 拼写练习 | `startSpellSession`, `submitSpellAnswer`, `getSpellResult` | 会话管理 |
| 阅读 | `getReadingPassages`, `getQuestions`, `submitReadingAnswers` | 阅读答题 |
| 语法 | `getGrammarTopics`, `getGrammarDetail` | 语法知识库 |
| 用户数据 | `getFavorites`, `toggleFavorite`, `getMistakeBook` | 收藏/错题管理 |
| 签到 | `checkin`, `getCheckinCalendar` | 每日签到 |
| TTS | `getTtsAudio` | 音频获取与缓存 |

### 2.4 用户认证流程

```
用户点击"微信登录" → 小程序wx.login获取code
    → 云函数authLogin用code换取openid
    → 查询/创建用户记录 → 返回登录态token
    → 前端存储token，后续请求携带
```

**前期免费策略下**：不强制绑定手机号，仅获取微信昵称和头像即可使用全部功能。

### 2.5 TTS音频方案

| 层级 | 策略 | 说明 |
|------|------|------|
| 第一层 | 本地缓存优先 | 已下载音频存在本地，不重复请求 |
| 第二层 | 云存储CDN | 已生成音频存云存储，通过CDN分发 |
| 第三层 | Edge-TTS生成 | 未缓存的单词调用Edge-TTS免费生成 |
| 降级 | 静默跳过 | 所有层级失败时静默处理，不阻断学习 |

## 三、词汇获取与数据方案

> **详细词汇数据方案**: 见 `vocabulary_data_plan.md`（1304行完整方案）

### 3.1 词汇数据来源策略

**核心组合方案**: ECDICT（主数据）+ COCA词频（分级依据）+ 有道API（补充缺口）

| 数据源 | 定位 | 覆盖量 | 费用 | 推荐指数 |
|--------|------|--------|------|----------|
| **ECDICT** | 主数据源 | 77万词条（含cet4/cet6标签） | 免费(MIT) | ★★★★★ |
| **COCA词频表** | 难度分级依据 | 6万lemma | 免费 | ★★★★ |
| **KyleBing词库** | 交叉验证 | CET-4+CET-6约5000词 | 免费 | ★★★★ |
| **有道智云API** | 补充例句/音标 | 全量 | 50元体验金 | ★★★ |
| **Edge-TTS** | 音频生成 | 全量英语单词 | 免费 | ★★★★★ |

### 3.2 CET-4词库构建

**目标**: 约4500词，覆盖四级考试核心词汇

```
ECDICT筛选cet4标签词 → 与KyleBing CET-4词表交叉验证
    → 数据清洗（去重/格式统一/编码统一）
    → COCA词频映射难度分级
    → 人工抽检审核（释义/例句/音标）
    → 最终入库约4200-4500词
```

### 3.3 CET-6词库构建（重叠词处理）

**CET-6共约5500词，其中约1000-1500词与CET-4重叠**。

| 类型 | 数量 | 处理策略 |
|------|------|----------|
| 重叠词（CET-4已有） | 1000-1500 | 保留但标记`is_in_cet4=true`，CET-6学习时默认跳过 |
| CET-6新增词 | 2000-2500 | 标记`cet6_new=true`，作为六级学习重点 |

### 3.4 难度分级模型

采用四维度综合评分公式：

```
difficulty_score = 0.4 * COCA词频分 + 0.15 * 单词长度分 + 0.2 * 词性复杂度 + 0.25 * 柯林斯星级
```

| 难度等级 | 分值区间 | 预估词数 | 能力要求 |
|----------|----------|----------|----------|
| Level 1 初级 | < 0.25 | ~1500 | 基础高频 |
| Level 2 中级 | 0.25-0.5 | ~1800 | 四级核心 |
| Level 3 高级 | 0.5-0.75 | ~1200 | 六级核心 |
| Level 4 专家 | >= 0.75 | ~1000 | 六级拔高 |

### 3.5 语法知识库

覆盖四六级核心语法考点，共约130+知识点，分8大类：

| 大类 | 知识点数量 | 示例 |
|------|-----------|------|
| 时态语态 | ~20 | 现在完成时、过去完成时、被动语态 |
| 从句 | ~25 | 定语从句、状语从句、名词性从句 |
| 非谓语动词 | ~18 | 不定式、动名词、分词 |
| 虚拟语气 | ~12 | if虚拟、建议命令虚拟、wish虚拟 |
| 情态动词 | ~10 | must/may/might表推测 |
| 倒装强调 | ~15 | 完全倒装、部分倒装、强调句型 |
| 词汇搭配 | ~20 | 固定搭配、短语动词 |
| 特殊结构 | ~12 | 独立主格、插入语、省略句 |

### 3.6 阅读题库

| 维度 | 规划 |
|------|------|
| 文章数量 | ~200篇（真题+模拟题） |
| 题型覆盖 | 细节理解、主旨大意、词义猜测、推理判断 |
| 每篇题目 | 5道题 |
| 来源 | 历年真题（核心）+ 模拟文章 |
| 难度分级 | 与词汇难度等级对应 |

### 3.7 数据质量目标

| 指标 | 目标值 | 达成方式 |
|------|--------|----------|
| 音标覆盖率 | ≥ 95% | ECDICT基础+API补充 |
| 例句覆盖率 | ≥ 80% | 自动抓取+人工审核 |
| 词性标注准确率 | ≥ 98% | ECDICT自带+校验规则 |
| 难度分级准确率 | ≥ 90% | 多维度公式+人工抽检 |
| 数据准备工期 | 6-8周 | 含清洗、标注、审核 |

## 五、用户交互设计方案

> **详细交互设计文档**: 见 `interaction_design.md`（1258行完整方案）

### 5.1 用户学习动线

#### 新用户首次使用
```
打开小程序 → 品牌页(1.5s) → 微信授权登录 → 选择CET-4/CET-6
    → 选择词库范围 → 设定每日目标(默认20词) → 首页仪表盘
    → 新手引导(3步蒙层) → 进入单词卡片学习
```

#### 每日学习常规流程
```
打开小程序 → 首页仪表盘 → 自动签到检测
    → [继续上次学习] / [今日新任务] / [选择模块]
    → 学习完成 → 结果页 → 分享/打卡
    → 查看统计 / 复习错题 / 退出
```

#### 薄弱点闭环
```
学习中答错/标记 → 自动加入错题本
    → 首页"薄弱点提醒"高亮 → 专项复习
    → 错题重新练习 → 连续答对2次 → 标记已掌握
    → 阶段目标达成 → 成就解锁 → 分享成就
```

### 5.2 单词卡片学习状态机

| 当前状态 | 触发事件 | 下一状态 | 系统反馈 |
|----------|----------|----------|----------|
| 初始化 | 进入页面 | 加载中 | 骨架屏 |
| 加载中 | 数据完成 | 正面展示 | 显示单词+音标 |
| 正面展示 | 点击/翻转 | 翻转动画中 | rotateY旋转300ms |
| 翻转动画中 | 动画完成 | 背面展示 | 显示释义+例句 |
| 背面展示 | 左滑/下一题 | 翻页动画中 | 卡片滑出+新卡片滑入 |
| 背面展示 | 点击发音 | 播放中 | 波形动画+音频播放 |
| 播放中 | 播放完成 | 背面展示 | 恢复默认状态 |
| 背面展示 | 点击收藏 | 已收藏 | 心形填充+微缩放 |
| 任意状态 | 退出 | 退出确认 | 确认弹窗(进度已保存) |
| 后台暂停 | 返回前台 | 恢复学习 | "欢迎回来"+当前卡片 |

### 5.3 翻页手势参数

| 参数 | 设定值 |
|------|--------|
| 触发阈值 | 水平滑动 > 60px |
| 灵敏度 | 速度 > 0.3px/ms 或 距离 > 60px |
| 垂直容忍 | 垂直偏移 < 40px（超则视为滚动） |
| 动画时长 | 280ms, ease-out |
| 边界反馈 | 首/尾卡片阻尼效果，最多偏移30px后回弹 |
| 防误触 | 翻转后200ms内不响应滑动 |

### 5.4 拼写练习交互

```
进入 → 展示题目（中文释义+词性+音标）
    → 用户输入 → 点击提交/回车
    → 正确：绿色√ + 1.5s后自动下一题
    → 错误：红色× + 显示正确答案 + 手动点击继续
    → 全部完成 → 结果页（正确率%+错题列表）
```

**输入细节**:
- 英文键盘，自动聚焦
- 大小写不敏感
- 输入长度=目标长度时可自动提交（可开关）
- 提供"提示"按钮显示首字母（使用提示不计入正确率）

### 5.5 阅读答题交互

**布局策略**: 文章上部55%滚动区 + 底部45%固定题目区，边读边答

| 交互 | 设计 |
|------|------|
| 文章展示 | 连续滚动，300-500词长度适中 |
| 题目切换 | 底部左右滑动或题号按钮切换 |
| 选项选择 | 单选，点击选中，再次点击可取消 |
| 标记不确定 | 题目右上角"标记"按钮，后续在答题卡中高亮 |
| 答题卡 | 网格展示所有题号，已完成/未做/标记状态一目了然 |
| 提交 | 未答完题提醒确认，提交后不可修改 |
| 结果页 | 得分动画 → 逐题解析 → 一键加入错题本 |

### 5.6 打卡与激励体系

#### 签到机制

| 项目 | 设计 |
|------|------|
| 触发时机 | 打开首页自动检测，未签到则弹出签到窗 |
| 签到方式 | 点击"签到"按钮 |
| 连续签到 | 火焰图标+连续天数展示 |
| 中断处理 | 连续天数归零，显示"重新来过"鼓励文案 |
| 日历视图 | 月度网格，已签到日期绿色高亮 |

#### 成就系统（规划12项）

| 类别 | 成就示例 | 触发条件 |
|------|----------|----------|
| 学习里程碑 | 初出茅庐 | 首次完成10词学习 |
| 学习里程碑 | 词汇达人 | 累计学习1000词 |
| 坚持打卡 | 一周新手 | 连续签到7天 |
| 坚持打卡 | 百日筑基 | 连续签到100天 |
| 答题成就 | 百发百中 | 单次练习正确率100% |
| 答题成就 | 阅读能手 | 累计完成50篇阅读 |

### 5.7 异常边界处理

| 场景 | 交互方案 |
|------|----------|
| 网络断开 | 本地缓存操作，恢复后批量同步；提示"当前为离线学习，数据将在联网后同步" |
| 音频播放失败 | 重试1次，仍失败则Toast"发音暂不可用"，不阻断学习 |
| 学习中断（接电话） | 自动保存当前进度，返回后恢复原位 |
| 进程被杀 | 下次打开从首页进入，显示"上次学习到第X词，继续？" |
| 多端同时学习 | 以最后一次操作时间为准，覆盖旧数据 |

## 八、风险评估与应对

### 8.1 技术风险

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|----------|
| 云开发性能瓶颈（查询慢） | 高 | 中 | 优化索引、前端缓存、分页加载 |
| TTS服务不稳定 | 中 | 中 | 多级fallback策略、本地缓存优先 |
| 小程序审核不通过 | 高 | 低 | 提前阅读审核规范、避免敏感内容 |
| 多机型兼容性问题 | 中 | 高 | 重点机型测试、rpx适配方案 |

### 8.2 数据风险

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|----------|
| 词汇数据质量不达标 | 高 | 中 | 建立质量检查流程、人工抽检、用户反馈纠词 |
| 版权问题（词库/文章） | 高 | 低 | 使用MIT开源数据、真题合理使用、标注来源 |
| 数据丢失 | 高 | 低 | 定期备份、云开发自动备份机制 |

### 8.3 业务风险

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|----------|
| 用户留存率低 | 高 | 中 | 完善激励机制、推送提醒、社群运营 |
| 竞品竞争 | 中 | 高 | 差异化定位（专注四六级、免费策略） |
| 后续商业化困难 | 中 | 中 | 积累用户后再探索付费点、视频课程变现 |

### 8.4 缓解措施汇总

1. **技术层面**: 采用成熟技术栈、做好错误监控、建立降级策略
2. **数据层面**: 多来源交叉验证、建立质量审核流程、预留用户反馈通道
3. **产品层面**: MVP快速验证、数据驱动迭代、重视用户反馈
4. **运营层面**: 前期免费获客、社群沉淀、口碑传播
