## 1. 词汇数据来源分析

### 1.1 开源词库资源调研

#### 1.1.1 核心开源词典：ECDICT

| 属性 | 详情 |
|------|------|
| 项目地址 | https://github.com/skywind3000/ECDICT |
| 许可证 | MIT License (可商用) |
| 词条数量 | 77万+ (基础版) / 425万 (Ultimate版) |
| 核心字段 | word, phonetic, definition, translation, pos, collins, oxford, tag, bnc, frq, exchange, detail, audio |
| 四六级标签 | tag字段包含 `cet4` / `cet6` 标记 |
| 词频数据 | BNC词频序 + COCA词频序 |
| 数据格式 | CSV (UTF-8), 支持转换到 SQLite / MySQL |

**评估**: ECDICT是此项目的**首选基础词库**。其77万词条覆盖远超四六级需求，tag字段直接标记了四六级归属，包含音标、中英释义、词性、词频、变形等完整字段，MIT协议允许商用。可从中直接筛选cet4/cet6标签的词构建核心词库。

| 属性 | 详情 |
|------|------|

| 格式 | TXT / JSON |
| 覆盖 | CET4, CET6, 考研, SAT |
| 特点 | 按考试类型分文件存储，JSON格式便于前端直接使用 |

**评估**: 可作为CET4/CET6词表的**交叉验证源**。与ECDICT的cet4/cet6标签做比对，确保核心词汇无遗漏。

### 2.3 词汇去重与关联策略

#### 2.3.1 去重规则

```sql
-- 同形异义词处理
-- 例: "bear" (n.熊 / v.承受) 视为同一个词条，多释义合并
-- 例: "wind" (/wɪnd/ 风 / /waɪnd/ 缠绕) 视为同一个词条，多音标注音

-- 大小写统一: 全转小写，专有名词保留首字母大写变体
-- 拼写变体: 美式/英式拼写统一映射 (color/colour -> 主词条 color, 标注英式变体)
```

#### 2.3.2 重叠词标记策略

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `is_in_cet4` | boolean | 是否在CET-4词表中 | true / false |
| `is_in_cet6` | boolean | 是否在CET-6词表中 | true / false |
| `cet6_new` | boolean | 是否CET-6新增(不在CET-4中) | true / false |
| `first_learn_level` | enum | 首次学习级别 | 'cet4' / 'cet6' / 'both' |
| `learned_status` | enum | 用户学习状态 | 'skipped' / 'learning' / 'mastered' |

**推荐策略**: 
- 重叠词(约1000-1500个)保留在CET-6表中，但UI默认显示"已学"标记

- 新增词(约2000-2500个)作为CET-6重点学习对象

### 2.4 词频分级方法

#### 2.4.1 基于COCA词频的4级难度划分

```
┌─────────────────────────────────────────────────────────────────┐
│                     词频分级映射表                               │
├──────────────┬─────────────────────┬──────────────┬────────────┤
│   难度等级    │   COCA词频排名       │   预估词数    │   能力要求  │
├──────────────┼─────────────────────┼──────────────┼────────────┤
│  初级(Level1) │   1 - 2000           │   ~1500      │   基础高频  │
│  中级(Level2) │   2001 - 6000        │   ~1800      │   四级核心  │
│  高级(Level3) │   6001 - 12000       │   ~1200      │   六级核心  │
│  专家(Level4) │   12001+ / 无词频    │   ~1000      │   六级拔高  │
└──────────────┴─────────────────────┴──────────────┴────────────┘
```

#### 2.4.2 难度综合评分公式

```
difficulty_score = w1 * coca_rank_score + w2 * word_length_score + w3 * pos_complexity + w4 * collins_stars

其中:
- coca_rank_score: COCA词频排名分 (排名越高=越难, 归一化0-1)
- word_length_score: 单词长度分 (长度越长=越难, 归一化0-1)  
- pos_complexity: 词性复杂度分 (词性越多=越难, eg: n./v./adj. = 复杂)
- collins_stars: 柯林斯星级 (0-5星, 星越低=越难)
- w1=0.4, w2=0.15, w3=0.2, w4=0.25 (权重可调)

最终映射:
- score < 0.25 -> Level 1 (初级)
- 0.25 <= score < 0.5 -> Level 2 (中级)
- 0.5 <= score < 0.75 -> Level 3 (高级)
- score >= 0.75 -> Level 4 (专家)
```

### 2.5 难度等级标注策略

| 指标 | 权重 | 说明 | 数据来源 |
|------|------|------|----------|
| COCA词频排名 | 40% | 词频越低=使用越少=越难 | COCA 60,000词频表 |
| 单词长度 | 15% | 字母越多越难记 | ECDICT word字段 |
| 词性复杂度 | 20% | 多词性=多用法=越难 | ECDICT pos字段 |
| 柯林斯星级 | 25% | 柯林斯词典标注的常用度 | ECDICT collins字段 |
| 考纲标注 | 辅助 | 官方考纲中的星标 | 考纲文件 |

#### 6.2.2 批量生成示例

```python
# 单词发音批量生成脚本
import edge_tts
import asyncio
import os

VOICE_US = "en-US-JennyNeural"
VOICE_UK = "en-GB-SoniaNeural"
OUTPUT_DIR = "./audio"

async def generate_word_audio(word: str):
    """为单个单词生成美式+英式发音"""
    word_lower = word.lower()
    
    # 美式发音
    us_path = f"{OUTPUT_DIR}/us/{word_lower[0]}/{word_lower}.mp3"
    os.makedirs(os.path.dirname(us_path), exist_ok=True)
    communicate = edge_tts.Communicate(word, VOICE_US)
    await communicate.save(us_path)
    
    # 英式发音
    uk_path = f"{OUTPUT_DIR}/uk/{word_lower[0]}/{word_lower}.mp3"
    os.makedirs(os.path.dirname(uk_path), exist_ok=True)
    communicate = edge_tts.Communicate(word, VOICE_UK)
    await communicate.save(uk_path)
    
    return {"us": us_path, "uk": uk_path}

async def batch_generate(words: list):
    """批量生成，带断点续传"""
    for i, word in enumerate(words):
        try:
            await generate_word_audio(word)
            print(f"[{i+1}/{len(words)}] {word} - OK")
        except Exception as e:
            print(f"[{i+1}/{len(words)}] {word} - FAILED: {e}")
            # 记录失败日志，后续重试
            with open(f"{OUTPUT_DIR}/failed.log", "a") as f:
                f.write(f"{word}\\n")

# 运行 (5500词预估生成时间: 约2-3小时)
```

```sql
-- 单词表
CREATE TABLE word (
    id INT PRIMARY KEY AUTO_INCREMENT,
    word VARCHAR(64) NOT NULL,
    word_variants JSON,
    phonetic_us VARCHAR(128),
    phonetic_uk VARCHAR(128),
    audio_url_us VARCHAR(256),
    audio_url_uk VARCHAR(256),
    syllable_count TINYINT,
    difficulty_level TINYINT NOT NULL DEFAULT 2,
    coca_rank INT,
    bnc_rank INT,
    collins_stars TINYINT,
    is_oxford_3000 BOOLEAN DEFAULT FALSE,
    word_length TINYINT NOT NULL,
    pos_count TINYINT NOT NULL DEFAULT 1,
    status TINYINT DEFAULT 0 COMMENT '0=正常 1=待审核 2=禁用',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_word (word)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 释义表
CREATE TABLE word_meaning (
    id INT PRIMARY KEY AUTO_INCREMENT,
    word_id INT NOT NULL,
    pos VARCHAR(16) NOT NULL,
    meaning_cn VARCHAR(512) NOT NULL,
    meaning_en VARCHAR(512),
    meaning_order TINYINT DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (word_id) REFERENCES word(id),
    INDEX idx_word_id (word_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 例句表
CREATE TABLE word_example (
    id INT PRIMARY KEY AUTO_INCREMENT,
    word_id INT NOT NULL,
    meaning_id INT,
    sentence_en VARCHAR(512) NOT NULL,
    sentence_cn VARCHAR(512),
    sentence_source VARCHAR(64),
    audio_url VARCHAR(256),
    example_order TINYINT DEFAULT 1,
    FOREIGN KEY (word_id) REFERENCES word(id),
    INDEX idx_word_id (word_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 词库表
CREATE TABLE word_bank (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(64) NOT NULL,
    code VARCHAR(16) NOT NULL UNIQUE,
    description VARCHAR(256),
    category VARCHAR(32) NOT NULL COMMENT 'exam/theme/grade',
    word_count INT DEFAULT 0,
    tags JSON,
    is_hot BOOLEAN DEFAULT FALSE,
    cover_image VARCHAR(256),
    status TINYINT DEFAULT 0,
    sort_order INT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 单词-词库关系表
CREATE TABLE word_bank_relation (
    id INT PRIMARY KEY AUTO_INCREMENT,
    word_id INT NOT NULL,
    bank_id INT NOT NULL,
    is_cet4_overlap BOOLEAN DEFAULT FALSE,
    learned_status TINYINT DEFAULT 0 COMMENT '0=未学 1=学习中 2=已掌握',
    review_count INT DEFAULT 0,
    last_review_at DATETIME,
    next_review_at DATETIME,
    mistake_count INT DEFAULT 0,
    FOREIGN KEY (word_id) REFERENCES word(id),
    FOREIGN KEY (bank_id) REFERENCES word_bank(id),
    UNIQUE KEY uk_word_bank (word_id, bank_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```
