## 1. 设计原则与风格定位

### 1.1 产品气质描述

本小程序面向**18-25岁大学生群体**，以英语四六级备考为核心场景。产品设计需要传递以下气质：

- **专业可信**：教育类产品需要让用户感到内容可靠、值得信赖，避免过度花哨或娱乐化的设计
- **高效专注**：学习场景要求界面不分散注意力，信息层级清晰，操作路径短
- **温暖有动力**：备考过程枯燥且压力大，界面需要通过色彩和交互给予正向反馈，让用户感到"被鼓励"
- **年轻活力**：面向大学生群体，设计语言不过于沉闷，有适度的现代感和精致度

### 1.2 设计关键词

| 关键词 | 设计表达 |
|--------|----------|
| **简洁** | 留白充分，每屏信息密度适中，去除装饰性元素 |
| **专注** | 学习页面沉浸式布局，隐藏非必要控件，减少干扰 |
| **温暖** | 主色调选用温暖绿色系（而非冷蓝），搭配柔和的圆角和阴影 |
| **成就感** | 数据可视化、打卡标记、进度展示、正确率反馈，处处体现成长感 |
| **清晰** | 信息层级分明，标题/正文/辅助文字对比度合理，一眼可找到重点 |

### 1.3 与微信设计规范的融合点

本设计严格遵循《微信小程序设计指南》的核心规范，在以下方面与微信生态保持一致：

- **导航规范**：顶部导航栏高度固定 44px，使用微信原生导航栏，不自定义；页面标题居中，字号 36rpx，字重 500
- **底部Tab栏**：5个Tab入口，高度 100rpx（含安全区），与微信原生Tab栏视觉风格统一（选中态填充色，未选中态灰色线框）
- **配色对齐**：主色选用与微信品牌绿相近的色调，确保小程序在微信环境中不突兀
- **字体栈**：优先使用系统字体，不引入外部字体文件，保证加载速度和一致性
- **按钮规范**：主按钮高度 88rpx，圆角 16rpx，与微信标准按钮尺寸对齐
- **弹窗规范**：模态弹窗宽度 560rpx，圆角 24rpx，与微信官方弹窗样式一致
- **交互反馈**：按钮点击态透明度 0.7，与微信标准点击反馈一致

### 1.4 设计哲学

> "少即是多，学习需要专注。" 

- **减法设计**：每个页面只保留核心信息，次要高优入口不超过2个
- **即时反馈**：用户的每个学习动作（答对/答错/收藏/打卡）都应有视觉或触觉反馈
- **游戏化克制**：学习进度和成就适度可视化，但不过度游戏化（避免喧宾夺主）
- **一致性**：相同功能的组件在全端保持一致的视觉和交互表现

## 2. 色彩系统

### 2.1 设计理念

教育产品色彩心理学的应用：
- **绿色**作为主色：象征成长、进步、生命力，给学习者积极的心理暗示（"我在成长"）
- **低饱和度处理**：降低色彩纯度，减少视觉疲劳，适合长时间学习场景
- **暖色倾向**：整体色板偏暖，营造舒适不冰冷的学习氛围
- **中性色分层**：灰色系从浅到深有明确的用途分层，确保信息层级清晰

### 2.11 深色模式切换规则

- **跟随系统**：默认跟随系统深色/浅色模式设置
- **手动切换**：个人中心提供手动切换开关，优先级高于系统设置
- **切换时机**：切换时无闪烁过渡，所有颜色变量即时更新
- **深色模式设计要点**：
  - 不使用纯黑（`#000000`）作为背景，使用深灰（`#121212`）以减少对比刺眼
  - 卡片表面比底层背景略亮，营造层级感
  - 主色在深色模式下饱和度提升约15%，确保可读性
  - 阴影在深色模式下加深（不透明度更高），因为浅色阴影在深色背景上不可见
  - 图片区域在深色模式下可添加轻微暗色遮罩（`rgba(0,0,0,0.15)`）以保持视觉协调

## 3. 字体与排版

### 3.1 中文字体栈

```css
/* 中文基础字体栈 */
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", "Helvetica Neue", Helvetica, Arial, sans-serif;
```

| 优先级 | 字体 | 平台 | 说明 |
|--------|------|------|------|
| 1 | `-apple-system` | iOS | iOS系统字体（苹方） |
| 2 | `BlinkMacSystemFont` | macOS | macOS系统字体 |
| 3 | `"PingFang SC"` | iOS/macOS | 苹方，iOS主要中文字体 |
| 4 | `"Hiragino Sans GB"` | macOS | 冬青黑体备用 |
| 5 | `"Microsoft YaHei"` | Android/Windows | 微软雅黑，Android和Windows主要中文字体 |
| 6 | `"Helvetica Neue"` | 通用 | 拉丁字母 fallback |
| 7 | `sans-serif` | 通用 | 系统无衬线字体兜底 |

### 3.2 英文字体栈

```css
/* 英文正文/界面字体 */
font-family: "SF Pro Display", -apple-system, BlinkMacSystemFont, "Segoe UI", "Helvetica Neue", Arial, sans-serif;

/* 单词展示专用字体栈 — 使用无衬线字体确保字母形态清晰 */
font-family: "SF Pro Display", "Helvetica Neue", Arial, "PingFang SC", sans-serif;
```

**关于等宽字体用于单词展示的说明**：
- **不使用等宽字体**（如 Menlo、Courier）作为单词展示字体
- 原因：等宽字体在移动端的阅读体验较差，字母间距不自然，显得过于"代码化"
- 替代方案：使用 `SF Pro Display` 或 `"Helvetica Neue"` 等比例无衬线字体，字母形态现代清晰，符合年轻用户审美
- 单词字母之间保持自然间距，不额外增加字间距

### 3.3 音标显示字体方案

```css
/* 音标专用字体栈 */
font-family: "Segoe UI", "Lucida Grande", "Arial", sans-serif;
```

- 音标符号（如 /ə/ /θ/ /ð/ 等）优先使用 `Segoe UI`（iOS/Android均有较好支持）
- 字号比英文单词小2个阶（如单词40rpx，音标32rpx）
- 音标颜色使用次要文字色 `#495057`（浅色）/ `#B5B5BD`（深色）
- 音标前后保留斜杠符号 `/ /`，斜杠颜色与音标文字相同

- **英文内容行高**：1.5 - 1.7，英文需要稍大行高以保障阅读流畅性
- **例句行高**：1.7，包含中英文混合内容，需要更大行高
- **段落间距**：段落之间保持 `24rpx` 间距

### 4.3 安全区适配

```

│  状态栏高度（动态获取）        │  ← 状态栏: statusBarHeight
```

#### 具体安全区数值

| 区域 | 高度 | 获取方式 | 说明 |
|------|------|----------|------|
| 状态栏 | 动态 | `wx.getSystemInfoSync().statusBarHeight` | iOS约44-59px，Android约24-32px |
| 导航栏 | 88rpx（44px） | 固定 | 微信小程序原生导航栏 |
| Tab栏 | 100rpx（50px）+ 底部安全区 | 固定 + 动态 | 底部安全区通过 `env(safe-area-inset-bottom)` 获取 |
| 底部安全区 | 动态 | `wx.getSystemInfoSync().safeArea.bottom` | iPhone X+ 约34px（68rpx），其他0 |

#### 安全区适配代码规范

```css
/* Tab栏底部安全区处理 */

  height: 100rpx;
  padding-bottom: env(safe-area-inset-bottom);
  background-color: #FFFFFF;
}

/* 页面内容底部预留Tab栏高度 */
.page-content {
  padding-bottom: calc(100rpx + env(safe-area-inset-bottom));
}

/* 底部固定按钮的安全区处理 */
.bottom-fixed-button {
  padding-bottom: env(safe-area-inset-bottom);
  padding: 24rpx 32rpx calc(24rpx + env(safe-area-inset-bottom));
}
```

### 4.4 布局网格系统

小程序采用**弹性布局**（Flexbox），不使用严格的网格系统：

- **内容区宽度**：`100% - 64rpx`（左右各32rpx边距）
- **两列布局**：`gap: 24rpx`，每项宽度 `calc((100% - 24rpx) / 2)`
- **三列布局**：`gap: 16rpx`，每项宽度 `calc((100% - 32rpx) / 3)`
- **四列布局**：`gap: 16rpx`，每项宽度 `calc((100% - 48rpx) / 4)`

### 4.5 圆角规范

| Token | 数值 | 使用场景 |
|-------|------|----------|
| radius-sm | 8rpx | 小型标签、内联元素 |
| radius-md | 12rpx | 列表项、小型卡片 |
| radius-lg | 16rpx | 标准卡片、按钮、弹窗 |
| radius-xl | 24rpx | 大型卡片、底部面板 |
| radius-2xl | 32rpx | 首页数据卡片、特殊容器 |
| radius-full | 9999rpx | 圆形按钮、头像、Badge |

### 4.6 阴影规范

| 层级 | 阴影值 | 使用场景 |
|------|--------|----------|
| shadow-sm | `0 2rpx 8rpx rgba(0,0,0,0.04)` | 轻微凸起，如选中态列表项 |
| shadow-md | `0 4rpx 16rpx rgba(0,0,0,0.06)` | 标准卡片阴影 |
| shadow-lg | `0 8rpx 32rpx rgba(0,0,0,0.08)` | 弹窗、底部面板 |
| shadow-xl | `0 16rpx 48rpx rgba(0,0,0,0.12)` | 强调卡片、特殊浮层 |

深色模式下阴影不透明度提升50%，以弥补深色背景上阴影可见度不足的问题。

### 4.7 不同屏幕尺寸适配策略

```
屏幕宽度基准：375px（iPhone 6/7/8 逻辑像素）
rpx 换算：1px = 2rpx
```

| 设备类型 | 逻辑宽度 | 适配策略 |
|----------|----------|----------|
| 小屏（iPhone SE） | 375px | 标准基准，无额外处理 |
| 中屏（iPhone 12/13/14） | 390px | 内容自然扩展，保持边距比例 |
| 大屏（iPhone Pro Max） | 430px | 内容自然扩展，卡片可适当增宽 |
| 大屏（iPad 微信小程序） | 744px+ | 内容区最大宽度限制 750rpx，居中显示，两侧留白 |
| Android小屏 | 360px | rpx自动缩放，确保文字最小可读 |
| Android大屏 | 412-450px | 内容自然扩展 |

**最大内容宽度限制**：
```css
.container {
  max-width: 750rpx;
  margin: 0 auto;
}
```

### 4.8 布局原则总结

1. **一致性**：相同层级的元素保持相同的间距和尺寸
2. **亲近性**：相关元素靠近，不相关元素远离（通过间距体现信息分组）
3. **呼吸感**：页面留白不低于内容面积的20%，避免信息过载
4. **边界清晰**：卡片与背景之间通过阴影或边框清晰区分
5. **阅读节奏**：标题-内容-辅助信息的间距递减，引导阅读视线

## 7. 图标系统

### 7.1 图标风格

- **风格**：线性图标（Outline style），辅以少量面性图标（Filled style，用于选中态）
- **描边粗细**：2rpx（标准），端点和连接处使用圆角（`stroke-linecap: round`, `stroke-linejoin: round`）
- **设计原则**：
  - 简洁不复杂，单个图标路径节点不超过12个
  - 2px等宽描边，保证视觉一致性
  - 图标视觉居中，在相同尺寸的容器中保持视觉平衡
  - 直角和圆角混合使用，保持品牌识别度

## 8. 动效规范

### 8.1 动效设计原则

- **有目的的动效**：每个动画都有明确的功能目的（引导注意力、反馈状态、解释层级关系）
- **克制不花哨**：教育产品动效应简洁高效，时长控制在300ms以内
- **一致性**：相同类型的交互使用一致的动画参数
- **可感知但不干扰**：动画增强体验但不拖延操作流程

### 8.2 页面转场动画

| 转场类型 | 动画效果 | 时长 | 缓动函数 |
|----------|----------|------|----------|
| 普通页面push | 新页面从右侧滑入（`translateX(100%) → 0`） | 300ms | ease-out |
| 普通页面pop | 当前页面向右滑出（`translateX(0) → 100%`） | 300ms | ease-in |
| Tab切换 | 无转场动画，内容即时切换 | 0ms | - |
| 底部面板滑入 | 从底部滑入（`translateY(100%) → 0`） | 300ms | ease-out |
| 底部面板滑出 | 向下滑出（`translateY(0) → 100%`） | 250ms | ease-in |
| 弹窗出现 | scale(0.92) + opacity(0) → scale(1) + opacity(1) | 200ms | ease-out |
| 弹窗消失 | scale(1) + opacity(1) → scale(0.92) + opacity(0) | 150ms | ease-in |
| 遮罩层 | opacity(0) → opacity(1) | 200ms | ease |

### 8.3 卡片翻转动画

- **触发**：点击卡片任意位置
- **动画类型**：Y轴3D翻转
- **技术实现**：
  - 父容器：`perspective: 1000rpx; transform-style: preserve-3d;`
  - 正面/背面：`backface-visibility: hidden;`
  - 背面初始：`transform: rotateY(180deg);`

| 阶段 | 变换 | 时长 | 缓动 |
|------|------|------|------|
| 正面翻离 | `rotateY(0) → rotateY(90deg)` | 150ms | ease-in |
| 背面翻入 | `rotateY(90deg) → rotateY(180deg)` | 150ms | ease-out |
| 总时长 | - | 300ms | ease-in-out |

- 翻转期间卡片不可点击
- 翻转完成后恢复可点击状态

### 8.4 答题反馈动效

**答对反馈**：
- 选项背景瞬间变为成功浅色底（0ms）
- 选项圆圈缩放 `scale(1) → scale(1.15) → scale(1)`，200ms，ease-out
- 勾选图标出现：scale(0) → scale(1)，150ms，spring弹性缓动
- 屏幕中央上方显示"✓ 正确"浮动标签：
  - translateY(20rpx) + opacity(0) → translateY(0) + opacity(1)，200ms
  - 停留 800ms
  - opacity(1) → opacity(0)，150ms

**答错反馈**：
- 选项背景瞬间变为错误浅色底
- 卡片水平晃动：translateX 0 → -10rpx → 10rpx → -10rpx → 10rpx → 0，400ms
- 错误选项圆圈缩放同答对
- 屏幕中央上方显示"✗ 错误"浮动标签（同答对动画）

### 8.5 按钮点击反馈

| 按钮类型 | 反馈效果 | 时长 |
|----------|----------|------|
| 主要按钮 | 背景色变深 + scale(0.97) | 100ms |
| 次要按钮 | 背景色变深 + scale(0.97) | 100ms |
| 图标按钮 | scale(0.9) + 透明度 0.7 | 100ms |
| Tab项 | 图标 scale(1.08) | 150ms |
| 列表项 | 背景变为 `#F1F3F5` / `#1A1A1E` | 100ms |

- 松开后恢复原始状态，100ms
- 使用 `:active` 伪类或 `catchtouchstart/catchtouchend` 实现

### 8.6 数字增长动画

- **适用场景**：首页数据更新、学习完成统计、打卡天数变化
- **动画效果**：数字从旧值滚动到新值
- **参数**：
  - 时长：800ms
  - 缓动：ease-out
  - 使用 `requestAnimationFrame` 实现平滑滚动
  - 数字使用 `tabular-nums` 确保不跳动
- **大数字处理**：超过1000时显示为"1,280"格式

### 8.7 进度条动画

- **线性进度条**：
  - 宽度变化：300ms ease-out
  - 使用CSS `transition: width 300ms ease-out`

- **环形进度条**：
  - `stroke-dashoffset` 从 max 到目标值
  - 时长：800ms
  - 缓动：ease-out

### 8.8 列表加载动画

- **新项插入**：高度 0 + opacity 0 → 自动高度 + opacity 1，250ms ease-out
- **项删除**：opacity 1 + 自动高度 → opacity 0 + 高度 0，200ms ease-in
- **列表刷新**：顶部显示Loading指示器，旋转动画

### 8.9 签到动画

- **签到成功**：
  - 对应日期格子 scale(1) → scale(1.2) → scale(1)，300ms，spring弹性
  - 格子背景从透明渐变为成功色，200ms
  - 连续签到时，格子之间出现连接线动画（从左到右依次点亮）
  - 顶部显示"连续签到 X 天"浮动提示

### 8.10 触觉反馈建议（vibrateShort）

| 场景 | 反馈类型 | 说明 |
|------|----------|------|
| 按钮点击 | `light` | 轻触反馈 |
| 答对题目 | `medium` | 中等强度，正向确认 |
| 答错题目 | `heavy` | 较强反馈，错误提示 |
| 签到成功 | `medium` | 正向确认 |
| 收藏/取消收藏 | `light` | 轻触反馈 |

## 9. 适配方案

### 9.1 iPhone/Android 差异处理

| 差异项 | iPhone | Android | 处理方案 |
|--------|--------|---------|----------|
| 状态栏高度 | 44-59px | 24-32px | 动态获取 `statusBarHeight` |
| 底部安全区 | X+系列有（34px） | 通常无 | 动态获取 `safeArea.bottom` |
| 导航栏返回 | 左滑手势返回 | 物理返回键 | 均支持，页面需处理返回逻辑 |
| 字体渲染 | 苹方，字重丰富 | 思源黑体，字重较少 | 使用系统字体栈，不强制细字重 |
| 滚动回弹 | 有橡皮筋效果 | 无 | 接受平台差异，不模拟 |
| 点击延迟 | 无（已优化） | 部分有300ms延迟 | 使用 `catchtap` 替代 `bindtap` |
| 阴影渲染 | 性能较好 | 性能略差 | 简化阴影，必要时使用边框替代 |

### 9.2 不同屏幕尺寸适配

#### 小屏手机（iPhone SE / 小屏Android，≤375px逻辑宽）

- **字号微调**：Display 从 64rpx 调整为 56rpx
- **卡片内边距**：从 32rpx 调整为 24rpx
- **间距压缩**：区块间距从 48rpx 调整为 40rpx
- **Tab栏图标**：保持 48rpx，文字 Caption-XS 20rpx 不变
- **底部按钮高度**：保持 96rpx（保证触控面积）
- **单词卡片高度**：从 520rpx 调整为 480rpx

#### 中屏手机（iPhone 12/13/14，390-430px逻辑宽）

- 使用标准设计规范，无需特殊调整
- 内容自然填充稍宽的屏幕

#### 大屏手机（iPhone Pro Max / 大屏Android，>430px逻辑宽）

- 内容区最大宽度限制 750rpx（375px逻辑像素 × 2rpx）
- 超出部分两侧留白
- 保持组件触控面积不变
- 卡片可增加内容密度（如显示更多信息行）

#### iPad（微信小程序）

- **适配策略**：内容区最大宽度 750rpx，水平居中
- **侧边留白**：使用页面背景色填充两侧
- **不响应横屏**：固定竖屏显示
- **触控面积**：所有可点击元素保持最小 88×88rpx
- **字体字号**：保持不变（rpx自动缩放）

### 9.3 横屏适配策略

**本小程序不支持横屏模式。**

原因：
1. 学习类应用以竖屏阅读体验最佳
2. 单词卡片、文章阅读等场景竖屏布局已优化
3. 减少适配复杂度，保证开发效率

**技术实现**：
```json
// app.json
{
  "pageOrientation": "portrait"
}
```

若用户在横屏状态打开小程序，提示"请竖屏使用以获得最佳体验"。

### 9.4 刘海屏/灵动岛适配

- **顶部安全区**：通过 `wx.getSystemInfoSync().safeArea.top` 动态获取
- **状态栏高度**：`wx.getSystemInfoSync().statusBarHeight`
- **导航栏位置**：紧接状态栏下方
- **内容安全区**：确保关键信息不被刘海/灵动岛遮挡
- **全屏背景**：页面背景色延伸到整个屏幕（包括状态栏区域），内容在安全区内

### 9.5 底部手势条适配（iPhone X+）

- **底部安全区高度**：`env(safe-area-inset-bottom)`，通常为 68rpx（34px）
- **Tab栏处理**：`padding-bottom: env(safe-area-inset-bottom)`
- **底部固定按钮处理**：`padding-bottom: calc(24rpx + env(safe-area-inset-bottom))`
- **页面内容**：`padding-bottom` 需包含 Tab栏高度 + 安全区高度

### 9.6 字体大小适配（系统字体缩放）

- 微信小程序支持跟随系统字体大小设置
- 使用 `rpx` 单位可自动适配系统字体缩放
- **最小字号保证**：任何文字不小于 20rpx（避免不可读）
- **布局弹性**：文字可能换行时，容器高度使用自适应（不固定高度）
- **测试场景**：在系统设置中将字体调至最大，检查所有页面是否正常显示
