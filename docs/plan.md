# 英语四六级小程序 — 完整开发流程设计计划

## 项目目标
基于现有PRD文档，设计一套专攻CET-4/CET-6的小程序完整开发流程文档，核心聚焦：
1. 词汇获取方案（数据源、词库构建、更新机制）
2. UI设计方案（小程序视觉规范、页面设计）
3. 用户交互设计（学习流程、答题交互、打卡机制）
4. 技术架构方案（小程序云开发可行性、数据库、接口）

## Stage 1 — 并行方案设计（4路并行）

### 1.1 技术架构师
- 评估微信小程序云开发对项目的满足度
- 数据库设计（云开发数据库方案）
- 用户认证（微信登录 vs 自建账号）
- 云函数设计
- 存储方案（音频、图片）
- 限流、安全、费用评估
- 输出：技术架构文档 + 数据库Schema

### 1.2 词汇数据专家
- 四六级词汇数据来源分析（开源/商业/爬虫）
- 词库构建方案（CET-4约4500词，CET-6约5500词）
- 词汇数据字段设计（音标、释义、例句、音频等）
- 词频分级、难度标注策略
- 词库初始化与更新机制
- 语法知识库设计（配合词汇的语法点）
- 输出：词汇获取方案文档

### 1.3 UI设计师
- 小程序设计规范（基于微信设计规范）
- 色彩系统、字体规范、间距系统
- 核心页面设计：首页、词库页、学习卡片、答题页、个人中心
- 图标系统、空状态、加载状态
- 适配方案（不同手机屏幕）
- 输出：UI设计规范文档 + 核心页面线框/原型描述

### 1.4 交互设计师
- 用户学习动线设计（新用户→学习→练习→巩固）
- 单词学习交互（卡片翻转、滑动、收藏、进度保存）
- 阅读答题交互（题型设计、作答流程、错题回顾）
- 打卡与激励机制（签到、连续学习、成就）
- 生词本、收藏、错题本交互
- 搜索、筛选、导航交互
- 输出：交互设计文档 + 流程图描述

## Stage 2 — 整合与评审
- 主Agent整合4个专家输出为完整开发流程文档
- 检查各模块间一致性
- 补充开发里程碑、团队分工、风险评估
- 输出最终完整文档

## Stage 3 — 文档输出
- 加载docx技能
- 将最终Markdown文档转换为专业Word文档
- 输出最终交付物

## 关键约束
- 前期免费，微信登录即用
- 专攻四六级，暂不涉及GRE/雅思/小初高
- 小程序云开发优先
- 预留视频课程扩展接口
